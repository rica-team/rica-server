# 贡献指南

我们非常欢迎您的参与！我们希望让贡献 RiCA 变得尽可能简单和透明，无论是：

- 报告错误
- 讨论代码现状
- 提交修复
- 提出新功能建议
- 成为维护者

## 我们使用 Github Flow

我们使用 [GitHub Flow](https://guides.github.com/introduction/flow/index.html)，因此所有代码更改都通过 pull request 进行。

## 使用 Github 的 [issue tracker](https://github.com/rica-team/rica-server/issues) 报告错误

我们使用 GitHub issues 来跟踪公共错误。通过 [打开一个新 issue](https://github.com/rica-team/rica-server/issues/new) 来报告错误；就这么简单！

## 编写详细的错误报告

**优秀的错误报告** 通常包含：

- 快速摘要和/或背景
- 复现步骤
  - 具体一点！
  - 如果可以，提供示例代码。
- 您期望发生的事情
- 实际发生的事情
- 备注（可能包括您认为发生这种情况的原因，或者您尝试过但无效的方法）

## 许可证

通过贡献，您同意您的贡献将根据其 GPL-3.0 许可证进行许可。
